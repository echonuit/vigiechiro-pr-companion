# Analyse et conception - VigieChiro PR Companion

Ce dossier contient le **dossier de spécification opérationnel** du projet, produit par l'équipe pédagogique à partir de l'[expression du besoin](../Expression%20du%20besoin.md). Vous l'utilisez comme point d'entrée pour votre travail de développement.

## Sommaire

1. [Modèle conceptuel](Modèle%20conceptuel.md) - les entités métier, leurs relations, les règles, les glossaires
2. [Personas](Personas.md) - les 3 utilisateurs cibles de référence
3. [Parcours utilisateurs](Parcours%20utilisateurs/index.md) - les trajectoires d'usage principales (un fichier par parcours dans le sous-dossier)
4. [Story mapping](Story%20mapping/index.md) - les 8 épopées, stories INVEST et critères d'acceptation, avec estimations en étoiles (un fichier par épopée dans le sous-dossier)
5. [Périmètre MVP](Périmètre%20MVP.md) - le sous-ensemble à livrer en phase 1, justifié
6. [Planification](Planification.md) - le Gantt prospectif des sprints de développement
7. [Maquettes](Maquettes/README.md) - les wireframes basse fidélité des écrans clés

## Comment lire ce dossier

L'ordre indiqué ci-dessus est l'ordre de lecture recommandé : on commence par poser le **vocabulaire** et le **modèle de données** (modèle conceptuel) qui servira de référence partagée, puis on déroule **qui** (personas) → **ce qu'il fait** (parcours) → **unités livrables** (stories) → **arbitrage** (MVP) → **planification** → **maquettes**.

Si vous découvrez une incohérence ou si vous avez une meilleure idée, **discutez-en avec l'équipe pédagogique** avant de dévier de la spécification. Le dossier n'est pas un dogme, c'est un point de départ partagé.
